# Climate Data Services — Web Application

แอปพลิเคชันเว็บแบบ Full-Stack สำหรับประมวลผล จัดการ และแสดงผลชุดข้อมูลสภาพภูมิอากาศหลายมิติ (NetCDF) ระบบนี้เชื่อมโยงการวิเคราะห์ข้อมูลสภาพภูมิอากาศที่ซับซ้อนเข้ากับเครื่องมือแผนที่เชิงโต้ตอบและการวิเคราะห์ทางสถิติที่ใช้งานได้จริง

---

## ภาพรวม

ระบบให้บริการ Data Pipeline แบบครบวงจร ครอบคลุมตั้งแต่การนำเข้าไฟล์ดิบ การคำนวณดัชนี การแสดงผลเชิงพื้นที่ ไปจนถึงการควบคุมสิทธิ์การเข้าถึงตามบทบาท (Role-Based Access Control) ระบบได้รับการออกแบบให้รองรับไฟล์ NetCDF ขนาดใหญ่อย่างปลอดภัยผ่านการอัปโหลดแบบ Chunked และการประมวลผลเบื้องหลังแบบ Asynchronous

## คุณสมบัติหลัก

- **Chunked Data Ingestion** — รองรับการอัปโหลด NetCDF ขนาดใหญ่โดยไม่เกิดปัญหา RAM spike ผ่าน Streaming I/O และ Lazy Loading
- **Climate Index Computation** — คำนวณดัชนีมาตรฐาน (SPI, TXx, PRCPTOT และอื่น ๆ) ด้วย `xclim` พร้อมรองรับการตั้งค่า Baseline และการตรวจจับเหตุการณ์ภัยแล้งและน้ำท่วมจากดัชนี SPI
- **Geospatial Processing** — ตัดขอบเขตเชิงพื้นที่ตาม Shapefile พร้อมการคำนวณค่าเฉลี่ยแบบ Area-Weighted Mean
- **Interactive Visualization** — แผนที่ Grid เชิงพื้นที่ด้วย Leaflet + D3.js พร้อม Mann-Kendall & Theil-Sen Slope maps สำหรับแบบ Trend และกราฟ Time-series / Seasonal Cycle ด้วย Recharts
- **Role-Based Access Control** — บทบาท `viewer` และ `analyst` ควบคุมด้วย JWT ทั้งระดับ Route และ API Endpoint

## สถาปัตยกรรมระบบ

ระบบประกอบด้วย 3 Service ที่ทำงานในรูปแบบ Container และถูกจัดการผ่าน Docker Compose

![Architecture Diagram](images/Architecture.png)

### Backend Layers

| Layer | Directory | หน้าที่รับผิดชอบ |
|---|---|---|
| **API Routes** | `routes/` | รับ HTTP Request, ตรวจสอบ JWT และส่งต่องานไปยัง Background Task |
| **Services** | `services/` | Business Logic, จัดการ Async Task และ File I/O แบบประหยัด Memory |
| **Processing** | `processing/` | Scientific Engine — Clipping, คำนวณดัชนี `xclim`, Export GeoJSON/JSON |
| **Database** | `database/` | SQLAlchemy Model และการจัดการ Connection |

## เริ่มต้นใช้งาน

### สิ่งที่ต้องติดตั้งก่อน

- [Docker](https://docs.docker.com/get-docker/) และ [Docker Compose](https://docs.docker.com/compose/install/) (v2+)

### รันด้วย Docker Compose (แนะนำ)

```bash
# Clone โปรเจคมายังเครื่อง
git clone <repository-url>
cd my-app

# Build Image และเริ่มต้น Service ทั้งหมด (frontend, backend, database)
docker compose up --build -d

# ดู Log แบบ Real-time ของทุก Service
docker compose logs -f

# ดู Log เฉพาะ Service ที่ต้องการ (เช่น backend)
docker compose logs -f backend
```

เมื่อระบบพร้อมใช้งาน สามารถเข้าถึงได้ที่:

| Service | URL |
|---|---|
| Frontend (React App) | http://localhost:10002 |
| Backend API (Swagger UI) | http://localhost:10001/docs |
| PostgreSQL | `localhost:10003` |

```bash
# หยุดการทำงานทุก Service (เก็บข้อมูลไว้)
docker compose down

# หยุดและลบ Volume ทั้งหมด (รีเซ็ตฐานข้อมูลและไฟล์ทั้งหมด)
docker compose down -v

# Build ใหม่เฉพาะ Service ที่ต้องการโดยไม่รีสตาร์ท Service อื่น
docker compose up --build -d backend
```

### การ Mapping Port

กำหนดไว้ใน `docker-compose.yml`:

| Container | Port ภายใน Container | Port บนเครื่อง Host |
|---|---|---|
| `frontend` (Nginx) | 80 | **10002** |
| `backend` (FastAPI) | 8000 | **10001** |
| `db` (PostgreSQL) | 5432 | **10003** |

### Persistent Volumes

| Volume | Path ใน Container | เนื้อหา |
|---|---|---|
| `./backend/uploads` | `/app/uploads` | NetCDF และ Shapefile ที่อัปโหลดมา |
| `./backend/data` | `/app/data` | ชุดข้อมูลสภาพภูมิอากาศที่ผ่านการประมวลผลแล้ว |
| `./backend/output` | `/app/output` | GeoJSON maps, JSON time-series, `merged.nc` |
| `postgres_data` | `/var/lib/postgresql/data` | ข้อมูล PostgreSQL |

## การพัฒนาบนเครื่อง Local (ไม่ใช้ Docker)

สำหรับการพัฒนาแบบ Iterative ให้รันแต่ละ Service แยกกัน

### Backend

**ความต้องการ:** Python 3.11, PostgreSQL

```bash
cd backend

# สร้างและเปิดใช้งาน Virtual Environment
python -m venv venv
source venv/bin/activate        # macOS / Linux
# venv\Scripts\activate         # Windows

# ติดตั้ง Dependencies
pip install -r requirements.txt

# รัน Development Server
uvicorn main:app --reload --port 8000
```

เข้าถึง API Docs ได้ที่: http://localhost:8000/docs

### Frontend

**ความต้องการ:** Node.js 18+

```bash
cd frontend

npm install
npm run dev
```

เข้าถึงแอปได้ที่: http://localhost:5173

> **หมายเหตุ:** ในโหมด Local Development ให้ตรวจสอบว่า API Base URL ใน `src/services/api.js` ชี้ไปที่ `http://localhost:8000`
