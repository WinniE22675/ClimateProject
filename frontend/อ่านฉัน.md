# Frontend Architecture

ระบบส่วนหน้า (Frontend) สำหรับแพลตฟอร์มบริหารจัดการและแสดงผลข้อมูลสภาพภูมิอากาศ พัฒนาด้วย **React** (ผ่าน Vite) และใช้ **Tailwind CSS** ในการจัดการ UI รองรับการโต้ตอบกับผู้ใช้ทั้งการแสดงแผนที่เชิงพื้นที่ (Leaflet & D3.js), การพล็อตสถิติกราฟ (Recharts) และการควบคุมกระบวนการอัปโหลด/เตรียมข้อมูล NetCDF

### ภาพรวมสถาปัตยกรรม (Architecture Overview)
แผนภาพด้านล่างแสดงโครงสร้างสถาปัตยกรรมและการพึ่งพา (Dependencies) ระหว่างคอมโพเนนต์ในระบบ ช่วยให้เห็นภาพรวมการทำงานว่าหน้าเพจ (Pages) มีการเรียกใช้งานคอมโพเนนต์ย่อย (Sub-components), ตัวจัดการสถานะ (Context) และบริการเชื่อมต่อ (API Services) อย่างไร

## รายละเอียดโมดูล (Module Documentation)
ระบบส่วนหน้าถูกแบ่งออกเป็นโมดูลหลักตามกลุ่มฟีเจอร์การใช้งาน เพื่อให้ง่ายต่อการพัฒนาต่อยอดและบำรุงรักษา โดยมีรายละเอียดฟังก์ชันและลอจิกการทำงานของแต่ละส่วนดังต่อไปนี้:

## Application Root & Routing (src/App.jsx & src/main.jsx)
จุดเริ่มต้น (Entry Point) และศูนย์กลางการกำหนดเส้นทาง (Routing) ของระบบ `main.jsx` ผูกแอปพลิเคชัน React เข้ากับ DOM (Document Object Model) ส่วน `App.jsx` กำหนดเส้นทางผ่าน `react-router-dom` และจัดการสิทธิ์การเข้าถึงตามบทบาทผู้ใช้
### Functions
- **Layout Setup :** จัดการโครงสร้างหน้าเว็บหลัก โดยล็อกให้ Navbar แสดงอยู่ด้านบนเสมอ
- **Public Routes :** เปิดหน้า `/login` ไว้เป็นเส้นทางสาธารณะ
- **ProtectedRoute :** ล็อกสิทธิ์เข้าถึงหน้าเว็บ โดยหน้า `/` (Dashboard) เข้าได้ทั้ง 'viewer' และ 'analyst' ส่วน `/manipulate` และ `/process` เข้าได้เฉพาะ 'analyst'

## Navbar (src/components/Navbar.jsx)
แถบนำทางหลัก (Navigation Bar) ปรับเปลี่ยนหน้าตาและเมนูแบบไดนามิกตามบทบาทผู้ใช้งาน
### Functions
- **Auto-hide :** ไม่แสดงผลในหน้า `/login` และ `/register`
- **Role-Based Menu :** เมนู "Manipulate" และ "Process" แสดงเฉพาะผู้ใช้งานสิทธิ์ 'analyst'
- **getLinkClass :** ตรวจสอบหน้าปัจจุบันเพื่อเน้นสีเมนูที่กำลังใช้งาน (Active state)
- **handleLogout :** ล้างข้อมูลเซสชันและพาผู้ใช้กลับไปหน้าแรก

# Authentication Module
## Auth Context (src/contexts/AuthContext.jsx)
React Context สำหรับจัดการสถานะล็อกอินของทั้งระบบ เก็บ JWT Token, บทบาท (Role) และสถานะ เพื่อให้ส่วนต่างๆ ตรวจสอบสิทธิ์ได้ง่าย
### Functions
- **decodeTokenPayload :** ถอดรหัส Base64 ของ JWT เพื่อดึงข้อมูล ID และบทบาทผู้ใช้ โดยไม่ต้องพึ่งพาไลบรารีภายนอก
- **AuthProvider :** คอมโพเนนต์หลักที่ห่อหุ้มแอปพลิเคชันเพื่อกระจาย State
- **useEffect :** ตรวจสอบและกู้คืน Token ค้างใน `localStorage` ตอนเริ่มเปิดหน้าเว็บ
- **login :** บันทึก Token ลง `localStorage` อัปเดตสถานะและถอดรหัสเพื่อบันทึกบทบาทผู้ใช้
- **logout :** ล้างข้อมูลเซสชัน ลบ Token และรีเซ็ตสถานะทั้งหมด

## Login Page (src/pages/LoginPage.jsx)
หน้าต่างเข้าสู่ระบบ ให้ผู้ใช้กรอกอีเมลและรหัสผ่านเพื่อเข้าใช้งานแพลตฟอร์ม
### Functions
- **handleLogin :** ส่งอีเมลและรหัสผ่านไปที่ API หากสำเร็จจะนำ Token มาอัปเดตระบบและเปลี่ยนหน้าไปที่ Dashboard พร้อมจัดการแจ้งเตือน Error

## Register Page (src/pages/RegisterPage.jsx)
หน้า User Interface สำหรับสร้างบัญชีผู้ใช้ใหม่ รองรับการสมัครแบบแบ่งบทบาท โดยระดับ 'Analyst' จะต้องกรอกรหัสผู้ดูแลระบบเพื่อยืนยันสิทธิ์
### Functions
- **handleRegister :** ตรวจสอบรหัสผ่านซ้ำและ Admin Code จากนั้นส่งข้อมูลสร้างบัญชี เมื่อสำเร็จจะทำการล็อกอินอัตโนมัติและพาย้ายหน้าต่าง

## API Service (src/services/api.js)
ศูนย์กลางการเชื่อมต่อ API จัดการส่ง HTTP Request จัดการแนบ Header และควบคุมการแจ้ง Error ให้เป็นมาตรฐาน
### Functions
- **getAuthHeaders :** ดึง JWT Token สร้าง Header
- **apiFetch :** Wrapper ครอบคำสั่ง `fetch` ทำหน้าที่แนบ Header อัตโนมัติ จัดการส่ง `FormData` อย่างถูกต้อง และล้าง Token ทิ้งหากพบสถานะ 401 Unauthorized
- **authAPI & datasetAPI :** ชุดฟังก์ชันสำเร็จรูปสำหรับระบบสมาชิกและการทำงานของชุดข้อมูลสภาพภูมิอากาศ

## Protected Route (src/components/ProtectedRoute.jsx)
Route Guard ที่ทำหน้าที่ปกป้องหน้าเว็บ ตรวจสอบสถานะล็อกอินและบทบาทก่อนอนุญาตให้เข้าถึง
### Functions
- **Access Control :** ตรวจสอบสถานะจาก `AuthContext` หากไม่ได้ล็อกอินจะเตะกลับหน้า `/login` หากสิทธิ์บทบาทไม่เพียงพอจะส่งกลับไปที่หน้า Dashboard ทันที

# Dashboard Module
## Climate Dashboard (src/pages/ClimateDashboard.jsx)
หน้าหลักแสดงผลข้อมูล เป็นศูนย์กลางให้ผู้ใช้เลือกชุดข้อมูล พื้นที่ และดัชนี จัดการ State หลักทั้งหมดเพื่อส่งต่อ (Props) ให้คอมโพเนนต์แผนที่และกราฟ
### Functions
- **useEffect (activeDataset) :** ดึงไฟล์ `metadata.json` ของชุดข้อมูลเพื่ออัปเดตพื้นที่ทำงาน (Workspaces) ตัวแปร และขอบเขตปี
- **useEffect (country) :** ซิงค์การตั้งค่า Shapefile อัปเดตรายชื่อจังหวัด และแสดงรายการดัชนีเฉพาะของพื้นที่นั้น
- **handleApplyYearRange :** ตรวจสอบความถูกต้องของปี ป้องกันปีเริ่มต้นมากกว่าปีสิ้นสุด และปรับช่วงปีให้อยู่ในขอบเขตข้อมูลจริงก่อนอัปเดตการแสดงผล
- **fetchDatasetList :** ดึงรายชื่อชุดข้อมูลทั้งหมดจาก Backend ผ่าน `datasetAPI` เพื่อแสดงใน Dropdown

## Indices Viewer (src/components/IndicesViewer.jsx)
สร้างกราฟ (Timeseries) ไดนามิกด้วย `recharts` สลับประเภทกราฟอัตโนมัติระหว่าง Bar Chart (เหตุการณ์ SPI) และ Line Chart (ข้อมูลรายปี/ฤดูกาลปกติ) ตามดัชนีที่เลือก
### Functions
- **prepareSPIMonthlyData & assignSPIEventColors :** ประมวลผลข้อมูล SPI รายเดือน ระบายสีแดง (ภัยแล้ง) และสีน้ำเงิน (น้ำท่วม) เมื่อค่าความรุนแรงเกิน Threshold ติดต่อกัน 2 เดือนขึ้นไป
- **computeMovingAverage :** คำนวณค่าเฉลี่ยเคลื่อนที่ (ค่าเริ่มต้น 21 ปี) เพื่อลดความผันผวนและแสดงแนวโน้มระยะยาว (Trend)
- **Data Fetching (useEffect) :** ยิง Request โหลดข้อมูล JSON รายปี (Annual) และรายเดือน (Seasonal) จาก Backend พร้อมกัน 
- **Dynamic Rendering :** ดัชนี "SPI" แสดงเป็น Bar Chart พร้อมเส้น Threshold ส่วนดัชนีปกติแสดง Line Chart รายปีซ้อนเส้นค่าเฉลี่ยเคลื่อนที่ ควบคู่กับกราฟวัฏจักรฤดูกาล 12 เดือน

## Grid Map Viewer (src/components/GridMapViewer.jsx)
วาดแผนที่โต้ตอบผ่าน `react-leaflet` แปลงข้อมูลภูมิอากาศเป็นชั้นข้อมูลด้วย `D3.js ` รองรับโหมด Actual/Trend พร้อมระบบโหลดข้อมูลอัจฉริยะสั่ง Backend สร้างไฟล์แผนที่เมื่อไม่พบข้อมูล
### Functions
- **MapBoundsController :** คำนวณขอบเขตและซูมกล้องไปยังพื้นที่ที่เลือกโดยอัตโนมัติ
- **BoundaryLayer & CountryContextLayer :** วาดเส้นขอบเขตและหรี่สี (Dim) พื้นที่รอบนอกเพื่อขับเน้นพื้นที่เป้าหมาย
- **fetchAllMaps :** ดึงไฟล์ GeoJSON ลอจิกสำคัญ: หากพบ 404 จะยิงคำสั่ง `/maps/generate` สั่ง Backend ประมวลผลสร้างแผนที่ทันทีแล้วดึงข้อมูลใหม่
- **D3 Color Scaling :** คำนวณ Quantile (ตัดค่าโดด) สร้างเกณฑ์สี (Threshold) และใช้ชุดสี D3 ระบายลงกริดแผนที่
- **style & onEachFeature :** กำหนดสีกริดและผูก Popup แสดงสถิติเชิงลึก (ค่า, ความชัน, p-value)
- **SigPoint :** กรองพื้นที่ p-value < 0.05 แล้ววาดจุดสีดำทับเพื่อแสดงนัยสำคัญทางสถิติ

## Legend (src/components/Legend.jsx)
แถบสัญลักษณ์สี (Legend) แบบยืดหยุ่นตามขนาดหน้าจอ ปรับเปลี่ยนตัวเลขและรูปแบบตามข้อมูลแผนที่ปัจจุบัน
### Functions
- **ResizeObserver Effect :** ดักจับการเปลี่ยนขนาดหน้าต่าง เพื่อคำนวณและวาดให้พอดีหน้าจอเสมอ
- **formatLegendValue :** ปัดเศษทศนิยมอย่างเหมาะสม (ตัดทิ้งหากหน่วยเป็น "วัน", บังคับ 2-4 ตำแหน่งสำหรับค่า SPI)
- **SPI Event Logic :** หากแสดงแนวโน้มเหตุการณ์ SPI ระบบจะวาดกล่องสีแยกประเภท (แดง/เทา/น้ำเงิน) แทนแท่งไล่ระดับสี

# Manipulate Page Module
## Upload Dataset Page (src/pages/UploadDatasetPage.jsx)
โครงสร้างหน้าเพจสำหรับจัดการอัปโหลดข้อมูลดิบ จัดเตรียม Layout และมอบตรรกะการอัปโหลดให้ `DatasetManager` เป็นตัวจัดการ
### Functions
- **UploadDatasetPage :** เรนเดอร์โครงสร้างหน้าเว็บและเรียกใช้งาน `DatasetManager`

## Dataset Manager (src/components/DatasetManager.jsx)
แผงควบคุมหลักสำหรับจัดการชุดข้อมูล สลับช่องข้อมูล (Preset 1-4 / Shapefile) เรียกดูไฟล์ กำหนดขอบเขตข้อมูล (Scope) และสั่งเริ่มกระบวนการ
### Functions
- **fetchFiles :** ดึงรายชื่อไฟล์ที่อัปโหลดไว้จาก Backend ตามช่อง (Slot) ที่กำลังเลือก
- **handleDelete :** ยืนยันและส่งคำสั่งลบไฟล์ที่ระบุออกจากระบบ
- **handleProcessSelection :** ตรวจสอบฟอร์ม แปลงค่า Scope ว่างให้เป็น `null` และส่งคำสั่ง Backend รวมไฟล์และตัดขอบเขต

## Dataset Uploader (src/components/DatasetUploader.jsx)
คอมโพเนนต์จัดการการเลือกและอัปโหลดไฟล์ ป้องกันขนาดไฟล์เกินกำหนด และเตรียมข้อมูล `FormData` ส่งไปยังเซิร์ฟเวอร์
### Functions
- **handleUpload :** ตรวจสอบขนาดไฟล์ไม่ให้เกินกำหนด (1GB) เตรียมข้อมูล (Shapefile ส่งแบบ .zip เดี่ยว, NetCDF ส่งแบบอาร์เรย์) และอัปโหลดผ่าน API

# Process Page Module
## Dataset Process Page (src/pages/DatasetProcessPage.jsx)
ศูนย์ควบคุมการเตรียมข้อมูล แสดง Metadata โหลด/ลบชุดข้อมูล สั่งคำนวณดัชนีใหม่ และป้องกันการลบชุดข้อมูลสำคัญของระบบ
### Functions
- **useEffect (activeDataset) :** รีเซ็ตสถานะและเรียกฟังก์ชันดึงข้อมูลใหม่เมื่อเปลี่ยนชุดข้อมูล
- **fetchData :** ดึง Metadata จาก Backend มาแสดงผลตัวแปรและมิติต่างๆ
- **handleCalculateIndices :** รวบรวมการตั้งค่าและส่งคำสั่งให้ Backend คำนวณดัชนี ย้ายไปหน้า Dashboard เมื่อสำเร็จ
- **Status Polling (useEffect) :** เช็คสถานะการประมวลผลจาก Backend ทุก 2 วินาที จนกว่าจะ 'ready' หรือ 'error'
- **handleDeleteDataset & handleDeleteWorkspace :** ส่งคำสั่งลบข้อมูล พร้อมระบบป้องกันการลบชุดข้อมูลรวมของระบบ

## Download Section (src/components/DownloadSection.jsx)
จัดการการดาวน์โหลดไฟล์ NetCDF ที่ผ่านการรวม (Merge) และตัดขอบเขต (Clip) พร้อมตรวจสอบสถานะความพร้อมและป้องกันการกดดาวน์โหลดซ้ำซ้อน
### Functions
- **handleDownload :** ยิงคำสั่ง GET ไปที่ Backend นำข้อมูลแปลงเป็น Blob สร้าง URL ชั่วคราว และจำลองการคลิกลิงก์เพื่อบังคับเบราว์เซอร์ดาวน์โหลดไฟล์

## Dataset Preview (src/components/DatasetPreview.jsx)
แสดงผลรายละเอียด (Metadata) ของชุดข้อมูลที่ถูกรวมสำเร็จ เพื่อให้ผู้ใช้ตรวจสอบความถูกต้องก่อนคำนวณดัชนี
### Functions
- **Render Metadata :** จัดเรียง JSON Metadata แบ่งหมวดหมู่เป็น ตัวแปร, มิติข้อมูล, ขอบเขตพื้นที่, และช่วงเวลา ให้อ่านง่าย

## Indices Selector (src/components/IndicesSelector.jsx)
แผงตั้งค่าการคำนวณดัชนี ให้ผู้ใช้เลือกขอบเขต Shapefile, กำหนดปีฐาน (Baseline), ค่า SPI Threshold, และเลือกดัชนีชี้วัด
### Functions
- **useEffect (availableVars) :** จับกลุ่มตัวแปรดิบในไฟล์ คัดกรองและจับคู่กับดัชนีชี้วัดที่คำนวณได้
- **toggleSelect, selectAll, selectCategory :** จัดการติ๊กเลือก/ยกเลิก ดัชนีแบบรายตัว, แบบหมวดหมู่, หรือทั้งหมด
- **fetchColumns :** ดึงข้อมูลคอลัมน์จาก Shapefile ให้อัตโนมัติ เพื่อให้ผู้ใช้เลือกใช้ระบุพื้นที่ย่อย
- **onCalculate :** รวบรวมข้อมูลตั้งค่าทั้งหมดส่งต่อให้ Parent Component (`DatasetProcessPage`) สั่งการ Backend
